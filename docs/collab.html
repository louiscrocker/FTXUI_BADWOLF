Multi-cursor · Shared State · Real-time

Real-time Collaboration

Like tmux's synchronized panes, but smarter. Multiple terminals share a live TUI session — each peer has a named cursor, events broadcast in real time, zero external dependencies.

"The Rebellion coordinates operations across star systems — one terminal at a time."

Architecture

┌─ Terminal A ─────────────────────────────────────────────────┐
│  CollabClient("Alice")                                       │
│    ↓ SendEvent(KEY_PRESS)         ↑ OnRemoteEvent(callback)  │
└──────────────────────────────────────────────────────────────┘
         │  TCP   [len][JSON]                 ↑
         ▼                                   │
┌─ CollabServer ───────────────────────────────────────────────┐
│   port 7777                                                  │
│   AcceptLoop → one thread per peer                           │
│   BroadcastExcept(sender) → all other peers                  │
└──────────────────────────────────────────────────────────────┘
         │                                   │
         ▼  TCP   [len][JSON]                 │
┌─ Terminal B ─────────────────────────────────────────────────┐
│  CollabClient("Bob")                                         │
│    ↓ SendEvent(KEY_PRESS)   ↑ OnRemoteEvent → cursor overlay │
└──────────────────────────────────────────────────────────────┘
🔌

Wire Protocol

4-byte LE length-prefix + JSON payload. Same framing as NetworkReactive.

🧵

Thread per Peer

Server spawns one handler thread per connected peer. All threads are joinable.

🎨

6 Peer Colors

Red, Green, Yellow, Blue, Magenta, Cyan — each peer gets a unique color.

🆔

UUID-lite IDs

8-char hex from /dev/urandom. Collision probability ≈ 1 in 4 billion.

Getting Started

Server Side

#include "ftxui/ui/collab.hpp"
using namespace ftxui::ui;

// 1. Create and start the server
auto server = std::make_shared<CollabServer>(7777);
server->Start();

// 2. Embed status in your UI
auto status = Renderer([server]() {
    return server->StatusElement();
});

// 3. Show peer list sidebar
auto peers = CollabPeerList(server);

// When done:
server->Stop();

Client Side

#include "ftxui/ui/collab.hpp"
using namespace ftxui::ui;

// 1. Connect
auto client = std::make_shared<CollabClient>("127.0.0.1", 7777, "Alice");
if (!client->Connect()) {
    // handle connection failure
}

// 2. Wrap any component for automatic event broadcasting + cursor overlays
auto my_comp = /* your existing component */;
auto collab_comp = WithCollabSession(my_comp, client);

// 3. Or use the shared notepad directly
auto notepad = CollabNotepad(client);

// 4. React to remote events
client->OnRemoteEvent([](CollabEvent ev) {
    if (ev.type == CollabEvent::Type::STATE_SYNC) {
        // apply remote state
    }
});

// 5. Status bar
Element bar = CollabStatusBar(client);

// When done:
client->Disconnect();

Shared Notepad

The simplest collaboration primitive — all peers type into the same buffer.

// Every keystroke triggers a STATE_SYNC broadcast with the full text.
// Incoming STATE_SYNC events update the local Input buffer.
auto notepad = CollabNotepad(client);
RunFullscreen(notepad);

badwolf collab CLI Reference

collab_demo

Full collaboration demo with shared notepad, peer list, and event log. 

# Server mode (accepts peers)
./collab_demo --server [PORT]
./collab_demo --server 7777

# Client mode (connect and collaborate)
./collab_demo --client HOST[:PORT] --name NAME
./collab_demo --client 127.0.0.1 --name Alice
./collab_demo --client 192.168.1.5:7777 --name "Obi-Wan"

collab_dashboard

Shared ops dashboard — all peers see the same live metrics. Press Enter to highlight a metric for everyone. 

# Server mode (drives metrics updates)
./collab_dashboard --server [PORT]

# Client mode (observe + interact)
./collab_dashboard --client HOST[:PORT] --name NAME

# Controls
↑/↓   — select metric
Enter  — highlight selected metric for all peers
q      — quit

Use Cases

🐞

Pair Debugging

Two engineers share a terminal session. Both see cursor positions, both can type. Like pair programming, but for ops war rooms.

📊

Shared Dashboards

One terminal drives live metrics (CPU, memory, network). All connected peers see the same real-time view. Click-to-highlight works across all terminals.

🎓

Live Demos

Presenter runs the server. Audience connects as clients. Everyone follows along in real time — even on different machines.

⚔️

Ops War Rooms

The Rebellion coordinates star system operations across the galaxy. Multiple operators, one shared terminal view, zero confusion about who is doing what.

📝

Shared Runbooks

CollabNotepad gives a simple shared scratchpad for incident notes, checklists, and runbook steps — visible to everyone on the call.

🔬

Remote Inspection

Wrap any existing component with WithCollabSession to instantly add multi-user awareness — no redesign required.
