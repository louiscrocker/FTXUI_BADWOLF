What's included

Everything You Need to Ship Beautiful Terminal Apps

30+ high-level components, reactive state, live maps, 3D globe, and bindings for every language.

🏗️

High-Level API

The ftxui::ui namespace provides 30+ components: Form, DataTable, Layout, Dialog, Wizard, CommandPalette — all via builder chains.

📊

Braille Charts

LineChart, BarChart, ScatterPlot, Sparkline, Histogram — all rendered in high-res 2×4 braille dots. Adaptive sizing fills any terminal.

🗺️

GeoJSON Maps

Real NE110m Natural Earth data. Pan, zoom, mouse interaction, custom overlays, marker pins, and great-circle arcs between cities.

🌐

3D Spinning Globe

Orthographic projection rendered in braille. Animates at 20fps in a background thread. City overlays, rotation control, mouse pan.

⚛️

Reactive State

Reactive<T>, Computed<T>, ReactiveGroup — mutex-safe, re-entrant, triggers UI refresh automatically.

⚙️

Process Runner

Fork/exec child processes, stream stdout/stderr in real-time to a log panel, send SIGTERM. The backbone of ftgit and ftdb.

📝

Markdown Renderer

Full inline parser: bold, italic, code, headings, lists, links. Scrollable viewport. Perfect for help screens and README viewers.

📋

Virtual List

O(visible) rendering. 50,000 rows scroll as fast as 50. Custom row renderer, keyboard navigation, mouse scroll, selection callbacks.

📑

Sortable Table

Click column headers to sort, type to filter, paginate with keyboard. Generic over any struct via column definitions.

🖱️

Full Mouse Support

GeoMap drag & zoom, table row clicks, button hover, scroll wheel. Every component respects mouse events.

🔍

Debug Overlay

Press Ctrl+D to toggle: render time, event rate, component tree, memory usage.

🐍

Python Bindings

pip install pyftxui. GIL-safe with py::gil_scoped_release during render. Full API parity.

🔧

C API

libftxui-c — opaque handle types, language-agnostic. Caller-owns memory model. Use from any language with FFI.

🦀

Rust FFI

Type-safe RAII wrappers over the C API. Idiomatic Rust: Drop frees handles, builder pattern matches Rust conventions.

🚀

Killer Apps

Three reference apps: ftop (system monitor), ftgit (Git TUI), ftdb (SQLite browser). Ship-ready and open source.

🌐

WebAssembly

Compile any BadWolf app to WASM. Run terminal UIs in the browser with zero backend.

Installation

Up and Running in Under 5 Minutes

# CMakeLists.txt
cmake_minimum_required(VERSION 3.14)
project(myapp)

include(FetchContent)
FetchContent_Declare(ftxui
  GIT_REPOSITORY https://github.com/louiscrocker/FTXUI_BADWOLF
  GIT_TAG        main
)
FetchContent_MakeAvailable(ftxui)

add_executable(myapp main.cpp)
target_link_libraries(myapp PRIVATE
    ftxui::ui       # High-level API (includes dom + component + screen)
)
# Install with vcpkg
vcpkg install ftxui

# CMakeLists.txt
find_package(ftxui CONFIG REQUIRED)
target_link_libraries(myapp PRIVATE ftxui::ui)
// Download ftxui.hpp and drop it next to your main.cpp
// No CMake required — just compile:
// g++ -std=c++20 main.cpp -o myapp

#define FTXUI_IMPLEMENTATION
#include "ftxui.hpp"

using namespace ftxui::ui;

int main() {
    RunFullscreen(Text("Hello from FTXUI!"));
}
# Install Python bindings
pip install pyftxui

# main.py
import pyftxui as ui

app = ui.Layout.vertical([
    ui.text("Hello from pyftxui!"),
    ui.LineChart().series("cpu", data).build(),
])
ui.run_fullscreen(app)
