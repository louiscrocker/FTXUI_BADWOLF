Constraint Layout System

CSS-like sizing for terminal UIs. Width(Percent(50)), MinWidth(20), responsive breakpoints that adapt to actual terminal width — all implemented with a simple two-pass algorithm and no external solver.

Core Concepts

📐 SizeConstraint

Four unit types — Chars, Percent, Fill, Auto — resolved against available space with a two-pass algorithm. No external solver needed. 

Constraints c;
c.width  = SizeConstraint::Percent(50);
c.min_width = 20;
c.max_width = 60;
auto el = ConstraintBox(inner, c);

📱 Responsive Breakpoints

Bootstrap-inspired breakpoints evaluated at render time against Terminal::Size().dimx. The highest fitting breakpoint wins. 

auto el = Responsive()
  .XS(narrow_layout)   // 0+ cols
  .SM(medium_layout)   // 60+ cols
  .MD(wide_layout)     // 80+ cols
  .LG(full_layout)     // 120+ cols
  .Build();

🗂️ ConstraintGrid

Define column widths with any mix of constraint types. Children auto-wrap into rows. Supports col_span, gap control, and component cells. 

ConstraintGrid grid;
grid.Columns({
  SizeConstraint::Chars(20),
  SizeConstraint::Percent(30),
  SizeConstraint::Fill()
});
grid.Gap(1).Add(el1).Add(el2).Add(el3);
auto layout = grid.Build();

🔲 ConstraintRow / Column

Like hbox/vbox but with per-child constraint specs. Pass 1 resolves fixed/percent children; Pass 2 distributes remainder to Fill children. 

Constraints fill;
fill.width = SizeConstraint::Fill();

auto row = ConstraintRow({
  {sidebar,  {.width=SizeConstraint::Chars(24)}},
  {content,  fill},
  {panel,    {.width=SizeConstraint::Percent(25)}},
});

🎭 Stack & AspectRatio

Stack renders layers with dbox for overlay effects. AspectRatio constrains width and height accounting for the ~2:1 terminal cell aspect ratio. 

// Z-layered elements
auto overlay = Stack({background, foreground});

// 16:9 widescreen box
auto widescreen = AspectRatio(inner, 16, 9);

↔️ Center & Spacer

Centering helpers wrap elements in filler() pairs. FlexSpacer is a named alias for filler(). 

auto centered = CenterBoth(dialog);
auto left = CenterH(badge);
auto spaced = hbox({
  label, FlexSpacer(), value
});

Two-Pass Algorithm

No external constraint solver. The layout engine uses a simple O(n) two-pass algorithm:

  1. Pass 1 — Resolve all Chars and Percent children. Sum their widths and track the remainder.
  2. Pass 2 — Distribute the remaining space equally among all Fill and Auto children, with the leftover pixel going to the first children (no rounding waste).
  3. Clamp — Apply min_width / max_width after resolution.
  4. Render — Each child is decorated with size(WIDTH, EQUAL, resolved) before composing into hbox / vbox.

Responsive breakpoint selection runs in O(b) where b is the number of breakpoints, querying Terminal::Size().dimx once per render frame.
