🎙 Voice + Live Themes

Say "activate matrix" or "switch to enterprise" and watch your terminal UI transform with smooth animated theme transitions — powered by the same Whisper transcription engine as voice commands, no LLM required.

QUICK START
// Wrap any app with live voice theme switching
#include "ftxui/ui/voice_theme.hpp"
using namespace ftxui::ui;

auto ctrl = std::make_shared<VoiceThemeController>();
auto app  = WithLiveThemes(MyApp(), ctrl);
App::Fullscreen().Loop(app);

// Or just use the all-in-one showcase:
auto showcase = ThemeShowcase(/*auto_cycle=*/true);
App::Fullscreen().Loop(showcase);

Voice Commands

The engine recognises natural-language wake words and extracts the theme name. Articles ("the", "a", "an") and the trailing word "theme" are stripped automatically.

Single-word triggers

"activate lcars"
"apply the matrix theme"
"use enterprise"
"set dark"
"theme nord"

Multi-word triggers

"switch to imperial"
"change to dracula"
"switch to dark blue cyberpunk"
"change to warm sunset orange"

NL fallback

Unknown theme names are passed to NLThemeParser::FromDescription() — so any English description generates a matching palette on the fly.

"activate retro amber 80s"
"switch to neon pink synthwave"

Transition Animation

Every theme change — whether triggered by voice, button click, or code — is animated through a ThemeTransition. The transition smoothly interpolates every colour channel using the AnimationLoop at up to 30 fps, posting a Event::Custom repaint after each frame.

The default duration is 800 ms, configurable via VoiceThemeConfig::transition_duration. 

VoiceThemeConfig cfg;
cfg.transition_duration =
    std::chrono::milliseconds(400);
cfg.continuous_listening = true;

auto ctrl =
  std::make_shared<VoiceThemeController>(cfg);
ctrl->Start();

// Callback on each change
ctrl->OnThemeChanged([](const Theme& t,
                         const std::string& name) {
  // t is already applied globally
});

API Reference

VoiceThemeController

The primary controller. Ties together voice transcription and theme application. 

VoiceThemeController(VoiceThemeConfig cfg = {});

void Start();              // begin listening
void Stop();               // stop listening
bool IsListening() const;

void ApplyTheme(const Theme&);
void ApplyThemeByName(const std::string& name);
void ApplyThemeFromDescription(const std::string& desc);

int OnThemeChanged(fn<void(Theme, string)>);
int OnCommand(fn<void(VoiceThemeCommand)>);
void RemoveCallback(int id);

static VoiceThemeCommand ParseCommand(const std::string&);
static Theme NamedTheme(const std::string&);
static std::vector<std::string> AvailableThemeNames();

LiveThemeBar

Drop-in bottom bar showing mic status, current theme, colour swatches, last voice command, and a text input for direct theme entry. 

ftxui::Component LiveThemeBar(
    std::shared_ptr<VoiceThemeController> ctrl = nullptr,
    VoiceThemeConfig cfg = {});

ThemeStage

A self-contained preview panel — buttons, input, data table, status bar — that reads GetTheme() on every render, picking up live changes immediately with no extra wiring. 

ftxui::Component ThemeStage();

ThemeShowcase

A full-page component combining preset buttons and a ThemeStage, with optional auto-cycle mode that rotates through all presets on a timer. 

ftxui::Component ThemeShowcase(
    bool auto_cycle = false,
    std::chrono::milliseconds cycle_interval
        = std::chrono::seconds(5));

ReactiveTheme

Meyer's singleton that wraps SetTheme/GetTheme with listener callbacks and optional animation. 

ReactiveTheme& ReactiveTheme::Global();

const Theme& Get() const;
void Set(const Theme& t, bool animate = true);

int OnChange(fn<void(Theme)>);
void RemoveOnChange(int id);

ftxui::Component Bind(ftxui::Component c);

WithLiveThemes

Wraps any existing component with a LiveThemeBar at the bottom and intercepts the V key to toggle voice listening. 

ftxui::Component WithLiveThemes(
    ftxui::Component app,
    std::shared_ptr<VoiceThemeController> ctrl = nullptr,
    VoiceThemeConfig cfg = {});
