App Runners

Three modes to run your FTXUI application. All handle the event loop, terminal setup/teardown, and rendering.

RunFullscreen

void RunFullscreen(Component root);

Takes over the full terminal. Clears screen, hides cursor, enters raw mode. The recommended mode for most apps.

RunFitContent

void RunFitContent(Component root);

Resizes the terminal to fit the component's natural size. Useful for dialog-style apps that don't need full-screen.

RunLoop

void RunLoop(Component root);

Gives you manual control over the event loop. Use when you need to integrate with an existing event system.

#include <ftxui/ui/all.hpp>
using namespace ftxui::ui;

int main() {
    auto app = Layout::Vertical({ Text("Hello!") });

    RunFullscreen(app);      // Full terminal takeover
    // or: RunFitContent(app);  // Fit to content
    // or: RunLoop(app);         // Manual loop
}

WithDebugOverlay

Component WithDebugOverlay(Component inner);

Wraps any component with a debug overlay. Press Ctrl+D to toggle a panel showing:

  • Render time (last frame, average, worst)
  • Event rate (events/second)
  • Memory usage
  • Component tree depth
RunFullscreen(WithDebugOverlay(myApp));

Theme System

Global color palette applied to all components. Set once before creating components for consistent styling.

SetTheme / GetTheme

void SetTheme(Theme t); Theme GetTheme();

Theme struct

struct Theme { Color primary; // Main UI color Color accent; // Highlight / selection color Color error_color; // Error state Color warning_color; // Warning state Color success_color; // Success state Color text_muted; // Secondary / muted text Color border_color; // Box/border color Color text; // Primary text color Color background; // Background color };

Built-in Themes

namespace Theme { Theme Default(); Theme Dark(); Theme Light(); Theme Nord(); Theme Dracula(); Theme Monokai(); }
SetTheme(Theme::Nord());

// Or custom:
Theme t = Theme::Dark();
t.primary = Color::RGB(124, 58, 237);
SetTheme(t);

Layout

Compose components into vertical stacks, horizontal rows, or responsive grids. All layouts use the FTXUI flexbox-inspired size negotiation.

Methods

class Layout { static Component Vertical(std::vector<Component> children); static Component Horizontal(std::vector<Component> children); static Component Grid(int cols, std::vector<Component> children); };
auto ui = Layout::Vertical({
    header_component(),
    Layout::Horizontal({
        sidebar() | size(WIDTH, EQUAL, 30),
        main_content() | flex,
    }),
    status_bar(),
});

Form

Declarative form builder. Automatically handles tab navigation between fields, enter-to-submit, and escape-to-cancel.

Methods

class Form { Form& Title(std::string title); Form& Field(std::string label, std::string& binding); Form& Password(std::string label, std::string& binding); Form& OnSubmit(std::function<void()> callback); Form& OnCancel(std::function<void()> callback); Component Build(); };
std::string user, pass;
auto login = Form()
    .Title("Login")
    .Field("Username", user)
    .Password("Password", pass)
    .OnSubmit([&]{ authenticate(user, pass); })
    .Build();

DataTable<T>

Generic read-only data table. Template parameter T is your data type. Column definitions use lambdas — no reflection needed.

Methods

template<typename T> class DataTable { DataTable& Columns(std::vector<ColumnDef<T>> cols); DataTable& Rows(std::vector<T> rows); DataTable& OnSelect(std::function<void(T)> callback); Component Build(); };
struct Employee {
    int id;
    std::string name, dept;
    double salary;
};

auto t = DataTable<Employee>()
    .Columns({
        {"ID",     [](auto& e){ return std::to_string(e.id); }},
        {"Name",   [](auto& e){ return e.name; }},
        {"Dept",   [](auto& e){ return e.dept; }},
        {"Salary", [](auto& e){ return "$" + fmt("{:.0f}", e.salary); }},
    })
    .Rows(employees)
    .OnSelect([&](Employee e){
        showEmployeeDetail(e);
    })
    .Build();

Reactive<T>

Mutex-protected observable value. Subscribe to changes. Setting from any thread triggers a UI refresh event.

template<typename T> class Reactive { explicit Reactive(T initial_value); T Get() const; void Set(T value); void Subscribe(std::function<void(T)> listener); };

MakeComputed

Creates a derived Reactive that updates automatically when its source changes.

template<typename T, typename F> auto MakeComputed( std::shared_ptr<Reactive<T>> source, F transform_fn ) -> std::shared_ptr<Reactive<decltype(transform_fn(source->Get()))>>;
auto count  = std::make_shared<Reactive<int>>(0);
auto doubled = MakeComputed(count, [](int v){ return v*2; });
auto label   = MakeComputed(count, [](int v){
    return "Count: " + std::to_string(v);
});
