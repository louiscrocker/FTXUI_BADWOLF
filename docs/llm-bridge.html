Talk to your UI

Describe a terminal UI component in plain English. The LLM Bridge converts your words into a live BadWolf component — no external LLM required.

"show me a table of users" "plot server metrics chart" "create a form for registration" "show world map" "monitor server logs"
// One-liner: parse English → live component
auto comp = UIGenerator::GenerateFromText(
"show me a dashboard");

// Full pipeline
auto intent = NLParser::Parse("table of servers");
// → Intent{ type=TABLE, entity="servers" }
auto comp = UIGenerator::Generate(intent);

// With Ollama (if running at localhost:11434)
auto intent = LLMAdapter::QueryLLM("show users");

// Interactive REPL — type English, get live UI
Run(LLMRepl());

NLParser Keyword Reference

Intent Type Trigger Keywords Example Prompt
TABLE table, list, show, display, rows, columns "show me a table of users"
CHART chart, graph, plot, histogram, sparkline "plot server metrics"
FORM form, input, create, edit, add "create a form for registration"
DASHBOARD dashboard, overview, monitor "show me a dashboard overview"
MAP map, world, geo "display a world map"
LOG log, events, history "monitor server logs"
PROGRESS progress, loading, status "show loading progress"
UNKNOWN (no match) "xyzzy bloop"

Matching is case-insensitive. Entity extraction: noun after trigger verb ("show users", "of servers"). Field extraction: "with columns name, email". Option extraction: "in blue", "large size", "with border".

Setup: Ollama Integration

Upgrade from rule-based NLP to a real LLM for richer understanding. No API key required.

1

Install Ollama

Download from ollama.ai or via Homebrew:

brew install ollama
2

Pull a model

Any instruction-following model works. llama3 is recommended:

ollama pull llama3
3

Start the server

Ollama listens on localhost:11434 by default:

ollama serve
4

That's it

LLMAdapter::OllamaAvailable() auto-detects the running server. The bridge sends a structured system prompt that instructs the LLM to return JSON. If Ollama is not running, the bridge silently falls back to the built-in rule-based parser.

Setup: OpenAI Integration

1

Set your API key

export OPENAI_API_KEY="sk-..."
2

Check availability

if (LLMAdapter::OpenAIAvailable()) {
    // key is set — you can extend QueryLLM for HTTPS
}

OpenAIAvailable() checks for the OPENAI_API_KEY environment variable. The current LLM bridge routes HTTP traffic to Ollama (plain HTTP, no TLS dependency). For OpenAI HTTPS calls, extend LLMAdapter::QueryLLM() with a TLS-capable socket layer.

LLM System Prompt

LLMAdapter::SystemPrompt() teaches the LLM to return structured JSON: 

You are a UI generator assistant. The user will describe a terminal UI
component in plain English. You must respond with ONLY valid JSON in
this exact format (no markdown, no explanation):
{"type":"TABLE","entity":"users","fields":["name","email"],"options":{}}

Valid types: TABLE, CHART, FORM, DASHBOARD, MAP, LOG, PROGRESS, UNKNOWN.
Set entity to the data subject (e.g. "users", "servers").
Set fields to column names if mentioned, otherwise [].
Set options to {} unless color/size/border is specified.
Respond ONLY with the JSON object.

Code Examples

One-shot generation

// #include "ftxui/ui/llm_bridge.hpp"
// using namespace ftxui::ui;

auto comp = UIGenerator::GenerateFromText(
    "show me a dashboard");

App::TerminalOutput().Loop(comp);

Full pipeline

auto intent = NLParser::Parse(
    "show users with columns name, email");
// intent.type   == IntentType::TABLE
// intent.entity == "users"
// intent.fields == ["name","email"]

auto comp = UIGenerator::Generate(intent);
Run(comp);

Interactive REPL

// Full REPL: type English → live preview
SetTheme(Theme::Dark());
Run(LLMRepl());

Add "?" help to any component

auto my_table = DataTable<User>()
    .Column("Name", ...)
    .Data(&users)
    .Build();

// Attach a "?" explain button
auto with_help = WithLLMHelp(my_table,
    "User Table");

Run(with_help);

Multi-intent parsing

auto intents = NLParser::ParseMulti(
    "show users table and plot metrics");
// → [Intent{TABLE, "users"},
//     Intent{CHART, "metrics"}]

std::vector<Component> comps;
for (auto& i : intents)
    comps.push_back(UIGenerator::Generate(i));

Ollama LLM query

// Requires: ollama serve (localhost:11434)
if (LLMAdapter::OllamaAvailable()) {
    auto intent = LLMAdapter::QueryLLM(
        "show a sorted list of files",
        "llama3");
    auto comp = UIGenerator::Generate(intent);
    Run(comp);
} else {
    // auto-fallback to NLParser
    Run(UIGenerator::GenerateFromText(...));
}
