Quick Start

The badwolf generate CLI command produces a complete .cpp file from a plain-English description. No LLM required — built-in templates cover the most common app patterns out of the box. 

$ badwolf generate "kubernetes pod monitor"
🤖 BadWolf AI Code Generator
Description: kubernetes pod monitor
Output: generated_app.cpp

Ollama not found. Using built-in templates...
  Matched template: monitor
  Keywords matched: monitor, kubernetes, pod

✓ Generated: generated_app.cpp

# Build immediately:
$ g++ -std=c++17 generated_app.cpp \
      -lftxui_ui -lftxui_component -lftxui_dom -lftxui_screen \
      -o pod_monitor
$ ./pod_monitor

With Ollama (AI-powered generation):

$ brew install ollama && ollama pull llama3
$ ollama serve &
$ badwolf generate "kubernetes pod monitor with CPU gauges" k8s.cpp
🤖 BadWolf AI Code Generator
Using Ollama LLM...
✓ Generated: k8s.cpp

Build with:
  g++ -std=c++17 k8s.cpp -lftxui_ui -lftxui_component -lftxui_dom -lftxui_screen -o myapp

How It Works

Input description  "kubernetes pod monitor"
         │
         ▼
  LLM Available?
  ┌──────┴──────────────────┐
  │ No                      │ Yes (Ollama / OpenAI)
  ▼                         ▼
TemplateLibrary         CodeGenerator
  │  Keyword scoring         │  BuildSystemPrompt()
  │  FindBest("k8s pod")     │  BuildUserPrompt(req)
  │  → "monitor" template    │  QueryOllama(...)
  │                          │  ExtractCode(response)
  └──────────┬───────────────┘
             ▼
       CodeGenResult
         .code          ← complete .cpp
         .cmake_snippet ← CMakeLists.txt snippet
         .LooksValid()  ← sanity check
         .WriteToFile() ← save to disk

When LLM generation fails or Ollama is unreachable, the generator falls back to template matching automatically — no user action needed.

CodeGenStudio — In-App Generator

Embed the code generator directly inside your running BadWolf app using the CodeGenStudio component. Type a description, choose style and feature toggles, and get syntax-highlighted code instantly. 

┌─ 🤖 BadWolf Code Generator ─────────────────────────────────────────────────┐
│ Describe your app: [kubernetes pod monitor with real-time metrics_______]  │
│ Style: [Modern ▾]  [✓ Charts] [✓ Tables] [✓ Real-time] [_ Map]       │
│                                    [ Generate → ]  [From Template]         │
│─────────────────────────────────────────────────────────────────────────────│
│ ✓ Generated from template: monitor                          [Save As…]    │
│─────────────────────────────────────────────────────────────────────────────│
│ Generated Code (58 lines)                                                   │
│  1 │ #include <ftxui/ui.hpp>                                               │
│  2 │ using namespace ftxui;                                               │
│  3 │ using namespace ftxui::ui;                                           │
│  4 │                                                                         │
│  5 │ int main() {                                                          │
│  6 │   SetTheme(Theme::Dark());                                              │
│ …  │                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘

Embed CodeGenStudio in your app:

#include <ftxui/ui.hpp>
#include <ftxui/ui/codegen.hpp>
using namespace ftxui;
using namespace ftxui::ui;

int main() {
  SetTheme(Theme::Dark());

  auto studio = CodeGenStudio([](const CodeGenResult& r) {
    // called when code is generated
    r.WriteToFile("output.cpp");
  });

  ScreenInteractive::Fullscreen().Loop(studio);
  return 0;
}

SyntaxHighlighter

Minimal C++ syntax highlighter built entirely on standard string processing — no external dependencies. Recognises keywords, strings, comments, numbers, and preprocessor directives and maps them to FTXUI colour decorators. 

// Highlight a full source file
auto elements = SyntaxHighlighter::Highlight(source_code);

// Highlight a single line
auto el = SyntaxHighlighter::HighlightLine("int x = 42;");

// Tokenise to inspect individual tokens
auto tokens = SyntaxHighlighter::Tokenize("return nullptr;");
for (auto& tok : tokens) {
  // tok.text, tok.type (Keyword / String / Comment / Number / …)
}

API Reference

CodeGenRequest

struct CodeGenRequest { std::string description; std::string style = "modern"; … };
Configures a generation request. Key fields: description (required), style ("modern" / "minimal" / "matrix" / "lcars"), features, model, max_tokens.

CodeGenResult

bool CodeGenResult::LooksValid() const;
Returns true if .code is non-empty and references both ftxui and a main( function.
bool CodeGenResult::WriteToFile(const std::string& path) const;
Write .code to path. Returns true on success.

TemplateLibrary

static const CodeTemplate* TemplateLibrary::FindBest(const std::string& description);
Keyword-score all built-in templates against description. Returns the highest-scoring match (score ≥ 1), or nullptr.
static std::string TemplateLibrary::Fill(const CodeTemplate& tmpl, const std::map<std::string,std::string>& vars);
Replace {KEY} placeholders with values from vars. Unknown placeholders are left unchanged.

CodeGenerator

CodeGenResult CodeGenerator::Generate(const CodeGenRequest& req);
Synchronous generation. Uses Ollama if available, falls back to GenerateFromTemplate automatically.
void CodeGenerator::GenerateAsync(req, on_token, on_complete);
Runs generation on a background thread. on_token is called with each ~40-char chunk; on_complete with the final CodeGenResult.

CodeGenStudio()

ftxui::Component CodeGenStudio(std::function<void(const CodeGenResult&)> on_generated = {});
Full interactive code generation panel — description input, style/feature toggles, generate buttons, and syntax-highlighted preview.

CodePreview()

ftxui::Component CodePreview(const std::string& code, const std::string& language = "cpp");
Scrollable, line-numbered, syntax-highlighted code view. Arrow keys / j-k / PageUp-PageDown for navigation.
