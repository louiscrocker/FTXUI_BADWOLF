🔌 Plugin System

Load FTXUI components from .dylib / .so shared libraries at runtime — no app restart, no recompilation. Plugins expose a clean C ABI so any compatible shared library is a drop-in widget source.

dlopen / dlsym RTLD_LOCAL InProcessPlugin (testing) PluginManager UI Hot-reload

Architecture

  ┌─ Your Application ──────────────────────────────────────────────┐
  │                                                                  │
  │  PluginRegistry::Instance().Load("widgets/libchart.dylib");     │
  │                         │                                        │
  │      dlopen ────────────┘                                        │
  │      dlsym ──► badwolf_plugin_info()   → PluginInfo             │
  │      dlsym ──► badwolf_create_component() → Component*          │
  │      dlsym ──► badwolf_destroy_component()                       │
  │                                                                  │
  │  auto comp = reg.Create("ChartWidget", R"({"type":"line"})");   │
  │  // comp is a normal ftxui::Component — use it anywhere         │
  └──────────────────────────────────────────────────────────────────┘

Each plugin uses RTLD_LOCAL so its symbols never leak into the global namespace — multiple plugins can define the same internal helper names without conflict.

Writing a Plugin

1
Declare metadata with BADWOLF_PLUGIN_DEFINE
// hello_plugin.cpp #include "ftxui/ui/plugin.hpp" #include "ftxui/component/component.hpp" #include "ftxui/dom/elements.hpp" // Exports: badwolf_plugin_info(), badwolf_plugin_version() BADWOLF_PLUGIN_DEFINE("hello", "1.0.0", "Hello World plugin", "BadWolf Team", (std::vector<std::string>{"HelloWorld", "Greeting"}))
2
Implement the factory with BADWOLF_PLUGIN_FACTORY
BADWOLF_PLUGIN_FACTORY { std::string n(name); if (n == "HelloWorld") { return new ftxui::Component(ftxui::Renderer([] { return ftxui::vbox({ ftxui::text("🐺 Hello from BadWolf Plugin!") | ftxui::bold, ftxui::text("Dynamic loading works!") | ftxui::color(ftxui::Color::Green), }) | ftxui::border; })); } return nullptr; } BADWOLF_PLUGIN_DESTROY { delete c; }
3
Build as a shared library
# CMakeLists.txt add_library(hello_plugin SHARED hello_plugin.cpp) target_link_libraries(hello_plugin PRIVATE ui) if(APPLE) target_link_options(hello_plugin PRIVATE "-undefined" "dynamic_lookup") endif()
4
Load at runtime
auto& reg = ftxui::ui::PluginRegistry::Instance(); // Load from .dylib / .so auto handle = reg.Load("libhello_plugin.dylib"); if (!handle) { std::cerr << reg.last_error() << "\n"; } // Create a component from the plugin auto comp = reg.Create("HelloWorld");

PluginRegistry API

MethodDescription
Instance() Returns the Meyer's singleton PluginRegistry.
Load(path) dlopen a .so/.dylib, resolve symbols, return PluginHandle. Returns nullptr on failure; check last_error().
LoadDirectory(dir) Scan directory for *.dylib (macOS) / *.so (Linux), load each. Returns count of successes.
Unload(name) Unloads plugin by name, calling dlclose. Returns false if not found.
Plugins() Returns vector<shared_ptr<PluginHandle>> of all loaded plugins.
FindProvider(component) Returns the first plugin whose provides list contains component, or nullptr.
Create(component, params) Searches all plugins and creates the named component, passing params as a JSON string.
Reload(name) Hot-reload: unload then re-dlopen from the same path.
AddSearchPath(dir) Appends a directory to the search path list.
last_error() Human-readable string from the last failed Load() or Create().

InProcessPlugin — testing & prototyping

Register a component factory without building a shared library. Ideal for unit tests and rapid iteration.

ftxui::ui::InProcessPlugin my_plugin( {"demo", "1.0.0", "Demo", "Dev", {"MyWidget"}}, [](const std::string& name, const std::string& /*params*/) { return ftxui::Renderer([]{ return ftxui::text("hello from in-process"); }); }); my_plugin.Register(); auto comp = ftxui::ui::PluginRegistry::Instance().Create("MyWidget"); // use comp... my_plugin.Unregister();

Platform Notes

macOS

Uses .dylib extension. dlopen is in libSystem — no extra linker flags needed.

Linux

Uses .so extension. CMake automatically links -ldl to the ui target.

Windows

Load() returns nullptr with error "plugins not supported on this platform". InProcessPlugin works on all platforms.
