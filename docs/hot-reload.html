⚡ Dev Mode: Hot Reload

File watcher that detects source changes, recompiles, and hot-swaps components in a running app. Zero restart needed during development.

What's included

📂

FileWatcher

Polling-based file watcher using std::filesystem. Watches directories recursively for .cpp, .hpp, and .h changes. No OS-specific APIs — works identically on macOS, Linux, and Windows.

🔄

HotReloader

Wraps your component factory. On file change: debounce → detect → swap. The Component() wrapper always renders the latest factory result. Track reload count, last error, and status.

🎛️

DevApp

One-line dev runner. Adds a status bar showing reload count + current state. Pass your factory and watch paths; it handles the rest.

🎨

ReloadOverlay

Visual status indicator overlaid on your component: yellow dot while detecting, spinning icon while compiling, green tick on success, red bordered box on error.

🛡️

WithHotReload

In Release builds (NDEBUG), calls factory() directly. In Debug builds, wraps with full hot-reload support. Zero overhead in production.

🔥

badwolf dev

CLI command that builds your app, starts it with --dev, then polls for source changes and sends SIGUSR1 to trigger in-process reloads.

Usage Examples

Minimal DevApp

#include "ftxui/ui/hot_reload.hpp"

int main(int argc, char** argv) {
  bool dev = argc > 1 && std::string(argv[1]) == "--dev";

  auto factory = [] {
    return ftxui::Renderer([] {
      return ftxui::text("Hello, World!") | ftxui::border;
    });
  };

  if (dev) {
    ftxui::ui::DevApp(argc, argv, factory, {"src/", "include/"});
  } else {
    auto screen = ftxui::ScreenInteractive::Fullscreen();
    screen.Loop(factory());
  }
}

WithHotReload — transparent wrapper

// In Debug builds: full hot reload.
// In Release builds (NDEBUG): calls factory() directly, zero overhead.
auto comp = ftxui::ui::WithHotReload(
    [] { return MyComponent(); },
    { .watch_paths = {"src/"}, .debounce = std::chrono::milliseconds(200) }
);
screen.Loop(comp);

Manual FileWatcher with callbacks

ftxui::ui::FileWatcher watcher(std::chrono::milliseconds(100));
watcher.Watch("src/");
watcher.OnChange([](const std::string& path) {
    std::cout << "Changed: " << path << "\n";
});
watcher.Start();

// ... or poll synchronously:
auto changes = watcher.PollChanges();
for (auto& p : changes) { ... }

How it works

  1. FileWatcher spawns a background thread that polls std::filesystem::last_write_time() every 250 ms.
  2. Any modified .cpp / .hpp / .h file triggers registered callbacks.
  3. HotReloader debounces the signal (default 300 ms) then calls TriggerReload().
  4. Within-process reload: the factory function is re-invoked; the new component replaces the old one atomically.
  5. The Component() wrapper always delegates to the current component — the screen loop just keeps rendering.
  6. badwolf dev goes further: it rebuilds the binary on disk and sends SIGUSR1 to the running process.
No OS-specific APIs. Pure std::filesystem polling works on macOS, Linux, and Windows (C++17). No inotify, kqueue, or FSEvents required.
