🎙 Voice Control

Speak to your terminal app. Microphone → whisper.cpp transcription → NLParser → action. Both whisper.cpp and ffmpeg are optional — everything degrades gracefully to keyboard simulation mode.

🎙 Microphone   ──ffmpeg──►   WAV file   ──whisper──►   Transcription
                                                                ──NLParser──►   UIIntent
                                                                                   ──dispatch──►  handler(entity)

Setup

1 · Install ffmpeg optional

Used for microphone capture. On macOS it accesses AVFoundation; on Linux it uses ALSA.

# macOS
brew install ffmpeg
# Ubuntu
apt install ffmpeg

2 · Install whisper.cpp optional

Provides local speech-to-text. Download a model and build the CLI from source.

git clone https://github.com/ggerganov/whisper.cpp
cd whisper.cpp && make
./models/download-ggml-model.sh base

3 · Link the library required

Voice control is part of the ui library — no extra CMake option needed.

target_link_libraries(myapp PRIVATE
  ftxui::ui
)

4 · Include the header

Single include for everything, or include voice.hpp directly.

#include "ftxui/ui.hpp"
// or
#include "ftxui/ui/voice.hpp"

Quick Start

// Create the engine
auto engine = std::make_shared<VoiceCommandEngine>();

// Register intent handlers
engine->OnCommand("table", [&](const std::string& entity) { ShowTable(entity); });
engine->OnCommand("chart", [&](const std::string& entity) { ShowChart(entity); });
engine->OnCommand("unknown",[&](const std::string& text) { Log(text); });

// Echo every transcription to the log
engine->OnTranscription([&](const std::string& text) {
  log.push_front("🎙 " + text);
});

// Wrap your UI with a voice bar at the bottom (press V to record)
auto app = WithVoiceControl(my_component, engine);
App::Fullscreen().Loop(app);

Recognised Commands

The NLParser maps speech to UIIntent objects with an intent type and an entity string. You can also register arbitrary keyword handlers that match anywhere in the transcription.

Utterance (example) Intent type Entity
"show users table"tableusers
"plot server metrics"chartserver
"create a new user form"formuser
"show me a dashboard overview"dashboarddashboard
"display a world map"mapworld
"show server log events"logserver
"show loading progress"progressloading
"increment counter" (custom keyword)unknownfalls back to keyword match

API Reference

VoiceConfig

Configuration passed to all voice components.

FieldDefaultDescription
whisper_path"whisper"Path to the whisper CLI binary
model_path"ggml-base.bin"whisper model file path
language"en"Transcription language code
sample_rate16000Recording sample rate (Hz)
max_record_duration10000msAuto-stop after this many ms
silence_timeout1500msStop recording after silence
continuousfalseKeep listening after each command
echo_transcriptiontrueCall OnTranscription callback

VoiceCommandEngine

Ties together AudioCapture + WhisperTranscriber + NLParser dispatch.

VoiceCommandEngine engine;
engine.OnCommand("table", handler); // intent type → handler
engine.OnTranscription(echo_fn); // called with every transcription
engine.StartListening(); // begin recording
engine.StopListening(); // stop (thread-safe)
engine.ListenOnce(callback); // record one command, then stop
engine.status(); // → VoiceStatus enum
engine.last_transcription(); // last successful text

UI Components

// Mic button + VU meter + transcription display
auto bar = VoiceInputBar(engine, cfg);

// Wrap any component — press V to toggle listening
auto app = WithVoiceControl(my_comp, engine, cfg);

// Compact status icon for status bars
auto icon = VoiceStatusIndicator(engine->status(), level);

AudioCapture

Low-level microphone capture via ffmpeg. Use directly for custom pipelines.

AudioCapture cap(cfg);
cap.StartRecording("my_audio.wav"); // non-blocking
// ... do other work ...
cap.StopRecording();

// Or blocking:
std::string path = cap.RecordToFile(std::chrono::seconds(5));

// Check availability
bool ok = AudioCapture::MicrophoneAvailable();
float level = cap.CurrentLevel(); // 0–1 for VU meter

WhisperTranscriber

Runs the whisper CLI on a WAV file. Returns a TranscriptionResult.

WhisperTranscriber wt(cfg);
TranscriptionResult r = wt.Transcribe("audio.wav");
if (r.success) {
  std::cout << r.text << "\n";
} else {
  std::cerr << r.error << "\n";
}

// Check availability
bool ok = WhisperTranscriber::WhisperAvailable();
std::string v = WhisperTranscriber::WhisperVersion();

Graceful Degradation

Full stack ✓

ffmpeg + whisper.cpp both present → full voice control. VoiceStatus::Listening → Processing → Ready.

whisper missing

Audio is still captured but Transcribe() returns success=false. Status becomes Error with a descriptive message.

ffmpeg missing

MicrophoneAvailable() returns false. Recording silently no-ops. Use keyboard simulation mode (press K in voice_demo).

Neither available

All constructors succeed, all queries return safe defaults. Status stays Unavailable. No crashes, no aborts.
