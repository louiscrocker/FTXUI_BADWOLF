Terminal Video Player

Play video files, webcam feeds, and animated GIFs as braille or half-block art directly in your terminal — powered by an ffmpeg pipe with zero libav dependencies.

#include "ftxui/ui/video.hpp"
⣿

Braille Rendering

2×4 dot resolution per character cell — highest detail ASCII art without color overhead.

▀

Half-Block Color

▀ character with RGB24 foreground + background — near-pixel-perfect color fidelity.

📷

Webcam Feed

Live camera capture via avfoundation (macOS) or v4l2 (Linux).

🔗

Zero libav

All decoding via popen() + ffmpeg pipe. No libavcodec headers or link flags needed.

🔀 ffmpeg Pipeline

video.mp4 ──ffmpeg─→ rawvideo rgb24 ──pipe:1──→ ReadFrame() ──→ VideoFrame{pixels}
command: ffmpeg -i video.mp4 -f rawvideo -pix_fmt rgb24 -vf scale=W:H pipe:1

camera (avfoundation) ──ffmpeg─→ rawvideo rgb24 ──pipe:1──→ ReadFrame()
command: ffmpeg -f avfoundation -framerate 30 -video_size 640x480 -i "0:none" -f rawvideo -pix_fmt rgb24 pipe:1

The decode thread reads exactly width × height × 3 bytes per frame from the pipe, then sleeps 1/fps seconds before the next read. ffmpeg handles all codec complexity; FTXUI never links against libavcodec.

🎨 Render Mode Comparison

⣿ Braille

  • 2×4 dots per character cell
  • Monochrome (white on black)
  • Highest spatial resolution
  • Best for fast movement
  • Uses Canvas::DrawPoint()

▀ ColorBlock

  • 1×2 pixel-rows per character
  • Full RGB24 via fg + bg color
  • Best color fidelity
  • Great for stills & GIFs
  • Uses ▀ + Color::RGB()

▀ GrayBlock

  • Same layout as ColorBlock
  • Luminance-only grayscale
  • Lower bandwidth terminals
  • Good for B&W content
  • Uses BT.601 luma formula

⚡ Quick Start

Play a video file

#include "ftxui/ui/video.hpp"
#include "ftxui/component/app.hpp"
using namespace ftxui;
using namespace ftxui::ui;

int main() {
    auto player = VideoPlayerFromFile("movie.mp4");
    App::Fullscreen().Loop(player);
}

Live webcam feed

auto feed = CameraFeed("0");  // device index or path
App::Fullscreen().Loop(feed);

Animated GIF

auto gif = GifPlayer("animation.gif", /*loop=*/true);
App::FitComponent().Loop(gif);

Custom VideoPlayer with config

VideoPlayer::Config cfg;
cfg.mode        = VideoPlayer::RenderMode::ColorBlock;
cfg.loop        = true;
cfg.show_fps    = true;
cfg.playback_speed = 2.0;          // 2× fast-forward

auto source = std::make_shared<FfmpegVideoSource>("clip.mp4", 0.5);
auto player = std::make_shared<VideoPlayer>(source, cfg);
auto comp   = player->Build();     // returns ftxui::Component
player->Play();
App::Fullscreen().Loop(comp);

📖 API Reference

VideoFrame

Member Description
GrayAt(x, y) BT.601 luminance of pixel (x,y) → [0,255]
Scale(w, h) Nearest-neighbor resize; returns new VideoFrame
ToBraille(cols, rows) Render as braille Element (cols*2 × rows*4 dots)
ToColoredBlock(cols, rows) Render as ▀ half-block Element with full RGB24

FfmpegVideoSource

FfmpegVideoSource(path, scale) Construct; scale defaults to 1.0 (native resolution)
Open() Probe dimensions with ffprobe, open ffmpeg pipe
ReadFrame(out) Read one RGB24 frame from the pipe; false = EOF
FfmpegAvailable() Static: true if ffmpeg binary found in PATH
FfmpegVersion() Static: ffmpeg version string

VideoPlayer

Build() Returns interactive ftxui::Component with key bindings
Play() / Pause() / Stop() Playback control; Play() starts the decode thread
SetRenderMode(mode) Switch between Braille / ColorBlock / GrayBlock
CurrentTime() / FrameCount() Thread-safe read of playback position

Keyboard Bindings (Build() component)

L

Toggle Loop

Cycle Render Mode

💻 Platform Notes

macOS

Camera capture uses -f avfoundation. Device "0" is the built-in FaceTime camera. Install ffmpeg via Homebrew: brew install ffmpeg.

Linux

Camera capture uses -f v4l2. Device path is typically /dev/video0. Install: apt install ffmpeg.

ffmpeg is optional. All code compiles and runs without ffmpeg installed. When unavailable, the player component renders a friendly "ffmpeg not found" message instead of crashing.
