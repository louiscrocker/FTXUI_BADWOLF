AI-Powered Theming

Describe a theme in plain English, get a Theme struct back. Works instantly with the rule-based engine — no LLM required. Upgrades automatically to Ollama or OpenAI when available for precise RGB generation.

auto t = ftxui::ui::ThemeFromDescription("dark blue cyberpunk with orange accents");
ftxui::ui::SetTheme(t);

API Reference

One-liner

#include "ftxui/ui/nl_theme.hpp"
using namespace ftxui::ui;

// Synchronous — rule-based (always works, zero deps)
Theme t = ThemeFromDescription("dark blue cyberpunk with orange accents");
SetTheme(t);

// Async — upgrades to LLM if Ollama / OpenAI is available
ThemeFromDescriptionAsync("neon pink synthwave", [](Theme t) {
    SetTheme(t);
});

ThemePicker component

// Interactive picker: type a description, see live swatch preview
auto picker = ftxui::ui::ThemePicker([](Theme t) {
    SetTheme(t);
    // … rebuild your UI
});
App().Add(picker).Run();

Animated theme transitions

Theme from = GetTheme();
Theme to   = ThemeFromDescription("matrix hacker green");

ThemeTransition tr(from, to, std::chrono::milliseconds(600));
tr.Start();  // AnimationLoop drives SetTheme() each frame

// Manual interpolation
Theme mid = LerpTheme(from, to, 0.5f);

LLM upgrade

// Check if LLM is reachable
if (LLMThemeGenerator::IsAvailable()) {
    // Precise RGB from Ollama (falls back to rule-based if unavailable)
    Theme t = LLMThemeGenerator::Generate("deep space navy gold", "llama3");
}

// Generate variants
auto variants = LLMThemeGenerator::GenerateVariants("cyberpunk", 3);

How It Works

📝

1. Parse

NLThemeParser scans the description for mood, style, color, and modifier keywords.

🗺️

2. Map

ThemeDescription is mapped to a full Theme palette: primary, accent, text, borders, buttons, status colors.

🤖

3. Upgrade

If Ollama or OpenAI is reachable, a structured JSON prompt replaces rule-based RGB values with LLM-generated ones.

✨

4. Apply

SetTheme() activates the theme globally. ThemeTransition animates the switch using AnimationLoop.
