SQL & DataFrame as First-Class Citizens

SqlValue and DataFrame bring structured data directly into your terminal UI — filter, sort, aggregate, and render tables without writing boilerplate. Plug in optional SQLite for full query power, or stay dependency-free with the in-memory DataFrame engine.

Quick Start API Reference
🔢

SqlValue

Typed variant — NULL, INTEGER, REAL, TEXT, BLOB. Value semantics, zero allocations for scalars.

🗄️

SqliteDb (optional)

Thin RAII wrapper over SQLite3. Enabled with -DFTXUI_ENABLE_SQLITE=ON. Execute queries, bind parameters, iterate result rows.

📊

DataFrame

In-memory columnar table. Load from CSV, vectors, or query results. Filter, sort, slice, aggregate — pandas-style, no Python required.

⚡

ReactiveQuery (optional)

Re-runs a SQL query whenever bound parameters change. Subscribers and the UI refresh automatically. Requires BADWOLF_SQLITE.

🔄

ReactiveDataFrame

Reactive wrapper around DataFrame. Mutate the frame and all bound UI components re-render — no manual invalidation.

🖥️

DataFrameTable

Interactive terminal table component. Scrollable, sortable by column, and live-linked to a ReactiveDataFrame.

Quick Start

1. SqlValue Construction

#include "ftxui/ui/sql.hpp"
using namespace ftxui::ui;

// Typed constructors
SqlValue null_val;                          // NULL
SqlValue integer_val{42};                   // INTEGER
SqlValue real_val{3.14};                    // REAL
SqlValue text_val{std::string("hello")};    // TEXT
SqlValue blob_val{std::vector<uint8_t>{}}; // BLOB

// Type predicates
bool is_int = integer_val.is_integer();  // true
bool is_null = null_val.is_null();       // true

// Typed access — throws on mismatch
int64_t n = integer_val.as_integer();    // 42
double  r = real_val.as_real();          // 3.14
std::string t = text_val.as_text();      // "hello"

// Safe access — returns default on mismatch
int64_t safe = null_val.get_integer(0);  // 0

2. DataFrame — Load, Filter, Sort, Slice

#include "ftxui/ui/dataframe.hpp"
using namespace ftxui::ui;

// Load from CSV file
auto df = DataFrame::FromCSV("metrics.csv");

// Load from vectors
DataFrame df2;
df2.AddColumn("host",  std::vector<std::string>{"web-01","web-02","db-01"});
df2.AddColumn("cpu",   std::vector<double>{12.5, 87.3, 45.1});
df2.AddColumn("alive", std::vector<int64_t>{1, 1, 0});

// Filter rows (predicate on SqlValue)
auto busy = df2.Filter([](const DataFrame::Row& row) {
    return row.at("cpu").as_real() > 50.0;
});

// Sort by column (ascending)
auto sorted = df2.SortBy("cpu", /*ascending=*/false);

// Take first N rows
auto top5 = sorted.Head(5);

// Select a subset of columns
auto slim = top5.Select({"host", "cpu"});

// Shape
std::size_t rows = df2.RowCount();   // 3
std::size_t cols = df2.ColCount();   // 3

3. DataFrame Aggregates

// Column statistics (numeric columns)
double total = df2.Sum("cpu");    // 144.9
double avg   = df2.Mean("cpu");   // 48.3
double lo    = df2.Min("cpu");    // 12.5
double hi    = df2.Max("cpu");    // 87.3
int64_t n    = df2.Count("cpu");  // 3 (non-null)

// Group-by aggregate
auto by_status = df2.GroupBy("alive").Mean("cpu");
// → DataFrame with columns: alive | cpu_mean

// Value counts
auto vc = df2.ValueCounts("alive");
// → DataFrame with columns: alive | count

4. ReactiveQuery — requires BADWOLF_SQLITE

#ifdef BADWOLF_SQLITE
#include "ftxui/ui/sqlite.hpp"
using namespace ftxui::ui;

// Open (or create) a database
auto db = std::make_shared<SqliteDb>("app.db");

// Execute DDL / DML directly
db->Execute("CREATE TABLE IF NOT EXISTS events "
             "(id INTEGER PRIMARY KEY, name TEXT, ts REAL)");
db->Execute("INSERT INTO events(name,ts) VALUES(?,?)",
            {SqlValue{"login"}, SqlValue{1700000000.0}});

// ReactiveQuery — re-runs on parameter change
auto query = std::make_shared<ReactiveQuery>(
    db,
    "SELECT name, ts FROM events WHERE ts > ? ORDER BY ts DESC LIMIT 20",
    /*params=*/std::vector<SqlValue>{SqlValue{0.0}});

// Subscribe to new result sets
query->OnResult([](const DataFrame& result) {
    // called whenever the query is re-executed
});

// Update a bound parameter — triggers re-execution + callbacks
query->SetParam(0, SqlValue{1700000000.0});
#endif

5. DataFrameTable Component

#include "ftxui/ui/dataframe_component.hpp"
using namespace ftxui::ui;

// Wrap a DataFrame in a reactive handle
auto rdf = std::make_shared<ReactiveDataFrame>(df2);

// Create a scrollable, sortable table component
auto table = DataFrameTable(rdf);

// Use inside any FTXUI layout
auto screen = ScreenInteractive::Fullscreen();
screen.Loop(table);

// Push new data — table re-renders automatically
rdf->Update([](DataFrame& frame) {
    frame.AppendRow({{"host", SqlValue{"web-03"}},
                     {"cpu",  SqlValue{22.0}},
                     {"alive",SqlValue{int64_t(1)}}});
});

API Reference

SqlValue

Typed SQL variant — NULL, INTEGER (int64), REAL (double), TEXT (string), BLOB (bytes). Value semantics — copy is always a deep clone.

Member Description
SqlValue()NULL value
SqlValue(int64_t / double / string / blob)Typed constructors
is_null / is_integer / is_real / is_text / is_blob()Type predicates
as_integer / as_real / as_text / as_blob()Typed access — throws on mismatch
get_integer / get_real / get_text(default)Safe access — returns default on mismatch
type()Returns SqlType enum
to_string()Human-readable representation
operator== / !=Deep equality (NULL == NULL)

SqliteDb — requires BADWOLF_SQLITE

RAII wrapper over a SQLite3 connection. The destructor closes the connection; move-only, not copyable.

Member Description
SqliteDb(path)Open/create on-disk database
SqliteDb(":memory:")In-memory database
Execute(sql, params)Run DDL / DML; bind optional params
Query(sql, params)Run SELECT — returns DataFrame
LastInsertRowId()Row ID of last INSERT
Changes()Rows affected by last DML
BeginTransaction() / Commit() / Rollback()Manual transaction control

DataFrame

In-memory columnar table. Columns are typed as SqlValue vectors. All transformation methods return a new DataFrame — the original is unchanged.

Member Description
DataFrame::FromCSV(path, opts)Parse CSV file — auto-detects types
AddColumn(name, values)Append a typed column
AppendRow(map)Add a row by column-name map
Filter(predicate)Keep rows matching predicate
SortBy(col, ascending)Sort rows by column value
Select(cols)Project subset of columns
Head(n) / Tail(n) / Slice(start, end)Row slicing
Sum / Mean / Min / Max / Count(col)Column aggregates
GroupBy(col).Mean/Sum/Count(col)Group-by aggregation
ValueCounts(col)Frequency table for a column
RowCount() / ColCount()Shape
ColumnNames()Ordered list of column names
At(row, col)Single cell access

ReactiveQuery — requires BADWOLF_SQLITE

Parameterised query that re-executes whenever a bound parameter changes, delivering a fresh DataFrame to subscribers and triggering UI refresh.

Member Description
ReactiveQuery(db, sql, params)Construct with initial parameter values
SetParam(index, val)Update a bound parameter — re-executes query
SetParams(vals)Replace all parameters atomically
OnResult(cb)Subscribe to new DataFrame results
Result()Read latest result synchronously
Refresh()Force re-execution without param change

ReactiveDataFrame

Reactive wrapper around DataFrame. Mutations via Update() notify all subscribers and trigger UI re-render.

Member Description
ReactiveDataFrame(df)Wrap an existing DataFrame
Get()Read current DataFrame (const)
Set(df)Replace entire DataFrame
Update(fn)Mutate in-place via callback — notifies
OnChange(cb)Subscribe to changes

CMake Integration

DataFrame only (no extra dependencies)

# DataFrame and SqlValue are always available
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build

# Link your target
target_link_libraries(my_app PRIVATE ftxui::component)

Enable optional SQLite support

# Optional SQLite support — enables SqliteDb and ReactiveQuery
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DFTXUI_ENABLE_SQLITE=ON
cmake --build build

# The macro BADWOLF_SQLITE is defined when SQLite is enabled
# Use it to guard SQLite-specific code:
#   #ifdef BADWOLF_SQLITE
#   #include "ftxui/ui/sqlite.hpp"
#   #endif

FetchContent (recommended for projects)

include(FetchContent)
FetchContent_Declare(ftxui
  GIT_REPOSITORY https://github.com/louiscrocker/FTXUI_BADWOLF
  GIT_TAG        main
)
set(FTXUI_ENABLE_SQLITE ON CACHE BOOL "" FORCE)  # optional
FetchContent_MakeAvailable(ftxui)

target_link_libraries(my_app PRIVATE ftxui::component)
