Component Registry

Install community-built widgets with a single command. Every component is a self-contained C++ header — no build step, no magic, just drop and include.

badwolf install sparkline-widget

⚡ Quick Start

# 1. Install the badwolf CLI (it lives in tools/) export PATH="$PATH:/path/to/FTXUI_BADWOLF/tools" # 2. Create a new project badwolf new my-dashboard # 3. Browse available components badwolf list # 4. Install a component badwolf install sparkline-widget # 5. Use it in your code #include "sparkline-widget/component.hpp"

Publish Your Component

Share your widget with the community in minutes. Every component is a single self-contained header — no dependencies, no build system changes.

1
Create your component header

Name it component.hpp. It must be self-contained (pragma once, only FTXUI includes). Put it in the namespace badwolf.

// BadWolf Component: my-widget v1.0.0 // Install: badwolf install my-widget #pragma once #include "ftxui/ui.hpp" namespace badwolf { ftxui::Element MyWidget(const std::string& text); }
2
Validate with badwolf publish

Run this in the directory containing your component.hpp:

badwolf publish
3
Fork, place, and open a PR

Fork the repo, add your file under registry/my-widget/component.hpp, add an entry to registry/index.json, and open a Pull Request. We review within 48 hours.

registry/ └── my-widget/ └── component.hpp

C++ Registry API

Self-register components at static init time and query the registry programmatically — perfect for plugin architectures.

#include "ftxui/ui/registry.hpp" // Auto-register at static init: BADWOLF_REGISTER_COMPONENT(my_button, "1.0.0", "A stylish button", []{ return ftxui::Button("Click me", []{}); }); // Query at runtime: auto& reg = ftxui::ui::Registry::Get(); for (const auto& m : reg.List()) { std::cout << m.name << " v" << m.version << "\n"; } auto results = reg.Search("button"); auto comp = reg.Create("my_button");
