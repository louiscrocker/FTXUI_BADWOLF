Terminal Game Engine

A pure-C++ 2D physics engine that renders on the BrailleCanvas at 8× resolution, plus a GameLoop component for building real terminal games — all without external libraries.

#include "ftxui/ui/physics.hpp"
⚙️

PhysicsWorld

Gravity, friction, restitution, circle/rect/line shapes, AABB collision detection.

🔁

GameLoop

Fixed-timestep loop integrated with AnimationLoop. OnUpdate / OnRender / OnInput callbacks.

🎨

PhysicsRenderer

Maps world coordinates to braille canvas dots. Fills terminal at flex resolution.

Quick Start

#include "ftxui/ui/physics.hpp"
#include "ftxui/component/screen_interactive.hpp"

using namespace ftxui;
using namespace ftxui::ui;

int main() {
  // One-liner built-in games
  auto pong      = Games::Pong();      // two-player pong
  auto breakout  = Games::Breakout();  // brick breaker
  auto gravity   = Games::Gravity();   // sandbox with Space to add balls
  auto billiards = Games::Billiards(); // pool/billiards demo

  // Custom physics world
  auto world = std::make_shared<PhysicsWorld>(Vec2{0, 9.8f});
  world->SetBounds(40.0f, 20.0f);
  world->AddCircle({20, 2}, 0.8f, 1.0f, Color::Cyan);
  world->AddStaticRect({20, 19}, 10.0f, 1.0f);  // floor

  PhysicsRenderer renderer(world, 40.0f, 20.0f);

  // Register with AnimationLoop for continuous physics stepping
  int cb = ui::OnFrame([&world](float dt) { world->Step(dt); });

  auto screen = ScreenInteractive::Fullscreen();
  auto comp = Renderer([&] { return renderer.Render(); });
  screen.Loop(comp);

  ui::RemoveFrameCallback(cb);
  return 0;
}

API Reference

Vec2

Lightweight 2D float vector. All math is inline — zero overhead.

Vec2 a{3, 4};
float len = a.length(); // 5.0
Vec2 n = a.normalized(); // {0.6, 0.8}
float d = a.dot(b); // dot product
Vec2 s = Vec2::Zero(); // {0, 0}

PhysicsWorld

Method Description
AddCircle(pos, r, mass, color) Add a dynamic circle. Returns body ID.
AddRect(pos, w, h, mass, color) Add a dynamic rectangle. Returns body ID.
AddStaticLine(p1, p2) Immovable line segment (mass=0).
AddStaticRect(pos, w, h) Immovable rectangle (mass=0).
Step(dt) Advance simulation by dt seconds. Call each frame.
SetBounds(w, h) Bodies bounce off a [0,w]×[0,h] boundary.
ApplyImpulse(id, impulse) Instant velocity change scaled by 1/mass.
OnCollision(fn) Register callback fired on every collision pair.
GetBody(id) Returns RigidBody* or nullptr if not found.

GameLoop

Integrates with AnimationLoop for fixed-timestep physics updates. Call Build() to get a renderable FTXUI Component. 

GameLoop loop(60);  // 60 FPS target

loop.OnUpdate([](const GameLoop::State& s) {
    // s.dt, s.frame, s.elapsed
    world->Step(s.dt);
});

loop.OnRender([&]() -> Element {
    return renderer.Render();
});

loop.OnInput([](Event e) -> bool {
    return false;  // not handled
});

auto comp = loop.Build();  // start + Component

Physics Pipeline

Each call to PhysicsWorld::Step(dt) runs these stages in order:

  1. Force integration — apply gravity + accumulated forces → velocity
  2. Position integration — velocity × dt → new position
  3. Damping — velocity × (1 − friction × dt)
  4. Boundary collision — reflect & clamp against world bounds
  5. Pair collision — circle–circle, circle–rect, rect–rect AABB
  6. Collision callbacks — notify registered OnCollision handlers
