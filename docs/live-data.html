Live Data Sources

FTXUI BadWolf provides powerful live data source adapters that automatically poll external data and update your UI in real-time. Build system monitors, log viewers, API dashboards, and more with minimal code.

Architecture

Source
→
Reactive<T>
→
Widget

Live sources run on background threads, polling data at configurable intervals. They integrate seamlessly with Reactive<T> for automatic UI updates.

Available Sources

FileTailSource

Watch files for new lines (like tail -f).

auto source = 
  std::make_shared<FileTailSource>(
    "app.log", 1000
  );
auto panel = LiveLogPanel(source);

HttpJsonSource

Poll HTTP endpoints for JSON data.

auto source = 
  std::make_shared<HttpJsonSource>(
    "api.example.com", 
    80, 
    "/status"
  );
auto viewer = LiveJsonViewer(source);

PrometheusSource

Scrape Prometheus metrics endpoints.

auto source = 
  std::make_shared<PrometheusSource>(
    "localhost", 
    9090, 
    "/metrics"
  );
auto table = LiveMetricsTable(source);

StdinSource

Non-blocking stdin reader for pipe input.

auto source = 
  std::make_shared<StdinSource>();
source->OnData([](const std::string& line) {
  // Process input
});

Custom Sources

Create your own by extending LiveSource<T>.

class MySource : public LiveSource<int> {
protected:
  int Fetch() override {
    return /* your logic */;
  }
};

WebSocketSource

Real-time WebSocket connections with automatic reconnection.

auto source = 
  std::make_shared<WebSocketSource>(
    "ws://server.com/feed"
  );

Quick Start Examples

File Tail Monitoring

auto source = std::make_shared<FileTailSource>("app.log");
auto panel = LiveLogPanel(source, 100);
RunFullscreen(panel);

HTTP API Dashboard

auto source = std::make_shared<HttpJsonSource>(
  "httpbin.org", 80, "/json", 
  std::chrono::seconds(5)
);
auto viewer = LiveJsonViewer(source);
RunFullscreen(viewer);

Prometheus Setup

Run a Prometheus node_exporter with Docker:

docker run -d -p 9100:9100 prom/node-exporter

Then connect your UI:

auto source = std::make_shared<PrometheusSource>(
  "localhost", 9100, "/metrics"
);
auto table = LiveMetricsTable(source);
RunFullscreen(table);

System Monitor

Create custom sources for system metrics:

class CpuSource : public LiveSource<double> {
protected:
  double Fetch() override {
    // Parse /proc/stat or run system commands
    return cpu_percentage;
  }
  
  std::chrono::milliseconds Interval() const override {
    return std::chrono::seconds(2);
  }
};

auto cpu = std::make_shared<CpuSource>();
auto chart = LiveLineChart(cpu, "CPU %", 60);

Widget Factories

Pre-built widgets that connect to live sources:

  • LiveLogPanel - Auto-scrolling log viewer
  • LiveMetricsTable - Formatted Prometheus metrics table
  • LiveLineChart - Sparkline with rolling history
  • LiveJsonViewer - Scrollable JSON response viewer

Thread Safety

All live sources are thread-safe:

  • Background polling runs on dedicated threads
  • Callbacks fire on the background thread
  • UI updates are triggered via App::PostEvent()
  • Mutex-protected state access
  • Graceful shutdown with Stop()
